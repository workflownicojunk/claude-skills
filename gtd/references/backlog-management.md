# GTD — Backlog Management

## Backlog Triage (wöchentlich, max. 20 Min.)

```
1. .claude/plans/backlog.md lesen
2. Alle Items nach 3 Fragen prüfen:
   → Ist es noch relevant? (nein → löschen)
   → Ist es actionable? (nein → Referenz oder Someday/Maybe)
   → Wann muss es erledigt sein? (→ Due Date setzen in ClickUp)
3. Top 3 für die Woche in ClickUp als High Priority markieren
```

## ClickUp Task-Kategorien (Domains)

```
Sales & Revenue     → Stripe-Issues, Pricing, Launches
Customer Support    → Offene Anfragen, Disputes, Beschwerden
Community           → Circle DMs, Posts, Events
Content             → Reels-Planung, Batch-Content
Tech & Automation   → n8n Workflows, Supabase, APIs
Finance             → DATEV, Steuer, SevDesk, Rechnungen
Strategy            → Planung, Roadmap, Partner
```

## Wann Tasks löschen vs. archivieren

| Situation | Aktion |
|-----------|--------|
| Task älter als 90 Tage ohne Fortschritt | Löschen |
| Task erledigt aber kein Outcome | Als Done markieren |
| Idee ohne konkreten nächsten Schritt | Someday/Maybe in backlog.md |
| Duplicate Task | Einen behalten, anderen löschen |
| Wird durch neuen Task ersetzt | Alten schließen mit Kommentar |

## Backlog-Gesundheit Metriken

```
Gesund:   < 20 aktive Tasks in ClickUp
Warnung:  20-35 Tasks → Triage-Session nötig
Kritisch: > 35 Tasks → Sofort-Triage, batch-löschen
```

**Faustregel:** Wenn du nicht alle offenen Tasks auswendig kennst, sind es zu viele.

## Someday/Maybe Kriterien

Tasks kommen in Someday/Maybe wenn:
- Keine klare Deadline existiert
- Noch nicht klar ob wirklich wertvoll
- Abhängig von Ressourcen die noch nicht da sind
- Gut aber nicht dringend und nicht wichtig

**Review:** Someday/Maybe Liste einmal im Monat prüfen. Entweder aktivieren oder dauerhaft löschen.
